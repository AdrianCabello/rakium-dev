# Design Standards

Build the actual usable experience as the first screen. Avoid generic SaaS hero filler.

Use:

- a clear visual concept tied to the client's domain
- strong typography with readable scale
- real content hierarchy
- responsive layouts with stable dimensions
- accessible contrast, focus states, and semantic landmarks
- tasteful motion for state changes, reveals, and navigation
- image assets when they improve trust or context

Avoid:

- lorem ipsum
- nested cards
- decorative gradient blobs
- one-color palettes
- oversized marketing UI for operational apps
- text that overflows buttons/cards on mobile
- hidden critical content behind animations

For Angular:

- prefer standalone components
- keep component state simple
- use signals where appropriate
- use lazy routes for larger apps
- keep dependencies lean
- use lucide icons when icons are needed
